普通中断原理

普通中断来说实现的原理较为简单，包括两个核心数组 g_irqvector[] 和 g_irqdata[] ，分别用来存储中断处理函数的函数指针和附加数据。对应具体操作如下：

  • 中断注册：

    1. 用户调用 aicos_request_irq()。

    2. 根据中断号将中断处理函数和附加数据存入 g_irqvector[] 和 g_irqdata[] 数组。

  • 中断处理：
    1. 中断发生后，所有中断首先会跳入 Default_IRQHandler() 通用中断处理函数中。
    2. 根据 MCAUSE 寄存器查询当前的中断号。
    3. 根据中断号在 g_irqvector[] 和 g_irqdata[] 数组中查询具体的中断处理函数指针和附加数据。
    4. 跳转到具体 ISR 中执行中断。

GPIO 中断原理

在 GPIO 中，一个 Group 只能共享一个中断且一个 Group 最多包含 32 个 Pin。然而，用户要求对每个 Pin 单独注册中断处理函数，因此需要对 GPIO Pin 二级中断进行特殊处理。这种设计既能实现功能又能让用户零感知。具体中断原理如下：

  • 扩大 g_irqvector[] 和 g_irqdata[] 数组，让每个 GPIO Pin 中断都拥有独立的存放位置，同时保持一级中断和二级中断格式一致。

  • 中断注册。在 RT-Thread 下，使用 rt_pin_attach_irq() 函数注册 GPIO Pin 中断，且使用 rt_pin_irq_enable() 函数使能 GPIO Pin 中断。

  • 处断处理。GPIO Pin 发生中断后，首先跳转到一级中断处理函数 drv_gpio_group_irqhandler() 中执行，通过查询发生中断的 Group 和 Pin Offset，最终在 g_irqvector[] 和 g_irqdata[] 数组中查询到具体的中断处理函数指针和附加数据。最后，到具体 ISR 中执行中断。

数据结构设计

中断模块关键结构体定义如下：
irq_handler_t/pin_irq_handler_t
enum irqreturn
{
    IRQ_NONE        = (0 << 0),
    IRQ_HANDLED     = (1 << 0),
    IRQ_WAKE_THREAD = (1 << 1),
};
typedef enum irqreturn irqreturn_t;
typedef irqreturn_t (*irq_handler_t)(int, void *);
typedef irqreturn_t (*pin_irq_handler_t)(void *);
