TSensor 配置

15 Jan 2024
Read time: 2 minute(s)

驱动配置

  1. 在 Luban-Lite 根目录下执行 scons --menuconfig，进入 menuconfig 的功能配置界面：
    scons --menuconfig
  2. 在 Board options 中，勾选 Using TSensor。
    Board options  --->
    [*] Using TSensor
    当使用 RT-Thread TSensor 驱动需要依赖 RT-Thread 的 Sensor 设备驱动框架，也是在 menuconfig 界面中打开：
    Rt-Thread options  --->
    RT-Thread Components  --->
        Device Drivers  --->
        [*] Using Sensor device drivers

    为了简化使用，Using TSensor 会自动打开 RT-Thread 的 Sensor 设备驱动框架。

  3. 进入 TSENS options 菜单，分别选中 [*] Using TSEN_CPU 和 [*] Using TSEN_GPAI。
    Board options  --->
    [*] Using TSensor
        TSENS options  --->
            [*] Using TSEN_CPU
            [*] Using TSEN_GPAI
  4. 配置 TSEN_CPU Parameter 和 TSEN_GPAI Parameter。
    Board options  --->
    [*] Using TSensor
        TSENS options  --->
            [*] Using TSEN_CPU
            [*] Using TSEN_GPAI
                TSEN_CPU Parameter  --->
                TSEN_GPAI Parameter  --->
                    obtaning data mode (single interrupt mode)
                        (X) single interrupt mode
                        ( ) polling mode

    TSensor 具有两种获取数据模式：

    1. 单次中断模式(single interrupt mode)，执行一次 test_tsen, 则执行一次温度读取

    2. 轮询模式 (polling mode)， 执行一次 test_tsen 后，根据轮询时间间隔进行轮询温度读取

  5. 返回上一级菜单，勾选 Using Efuse/SID。

    TSensor 校准功能依赖于 SID 获取校准参数，为此需打开 Using Efuse/SID。

    完整的配置输出示例如下：
    Board options  --->
    [*] Using TSensor
        TSENS options  --->
            [*] Using TSEN_CPU
            [*] Using TSEN_GPAI
                TSEN_CPU Parameter  --->
                TSEN_GPAI Parameter  --->
                    obtaning data mode (single interrupt mode)
                        (X) single interrupt mode
                        ( ) polling mode
    [*] Using Efuse/SID
注：

受 RTOS 环境的 API 接口限制，TSensor 控制器的高/低温告警、过温保护功能暂未提供。
